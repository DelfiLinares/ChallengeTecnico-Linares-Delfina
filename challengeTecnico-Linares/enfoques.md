# Dos enfoques para resolver el ejercicio

Para este ejercicio existían dos formas de trabajar: desarrollar el código de forma manual o utilizar inteligencia artificial como herramienta de apoyo. A continuación se explica cada una.

---

## Opcion 1: Sin IA (desarrollo manual)

El enfoque tradicional. El programador lee el enunciado, piensa la solución y escribe el código por su cuenta.

**Pasos del proceso:**

1. Leer el enunciado y entender qué se pide
2. Planificar la estructura del juego (variables, funciones, lógica)
3. Escribir el HTML, CSS y JavaScript
4. Probar en el navegador
5. Identificar errores y corregirlos
6. Repetir hasta que funcione correctamente

**Ventajas:**
- Se comprende en profundidad todo el código escrito
- Favorece el aprendizaje real de los conceptos
- Permite identificar errores con mayor precisión

**Desventajas:**
- Requiere más tiempo
- Puede ser frustrante cuando aparecen errores difíciles de resolver
- Exige conocimientos previos sólidos de HTML, CSS y JavaScript

---

## Opcion 2: Con IA

En lugar de escribir el código directamente, se le describe a la IA lo que se necesita y ella genera el código. El programador revisa, prueba y pide ajustes.

**Pasos del proceso:**

1. Leer el enunciado
2. Describirle a la IA qué tiene que hacer el programa
3. La IA genera el código
4. Probarlo en el navegador
5. Pedir modificaciones si algo no funciona o no es lo esperado
6. Repetir hasta obtener el resultado deseado

**Ventajas:**
- Es considerablemente más rápido
- Permite iterar sobre el resultado con facilidad
- No requiere recordar toda la sintaxis de memoria

**Desventajas:**
- Si no se entiende el código generado, es difícil detectar errores
- El resultado depende de qué tan bien se describe el problema
- A veces genera soluciones más complejas de lo necesario

---

## Comparación

Una de las diferencias más notorias entre ambos enfoques es el tiempo. Desarrollar el código de forma manual puede llevar varias horas, especialmente cuando aparecen errores que cuesta identificar. Con IA, el mismo resultado se puede obtener en minutos, ya que el código se genera automáticamente y los ajustes se hacen a través de indicaciones en lenguaje natural.

En cuanto al aprendizaje, el desarrollo manual tiene una ventaja clara. Al escribir cada línea uno mismo, se entiende qué hace cada parte del código y por qué. Con IA, en cambio, es fácil recibir un resultado sin comprender del todo cómo funciona, lo que puede ser un problema a largo plazo si el objetivo es aprender a programar.

El control sobre el código también varía bastante. Sin IA, el programador decide exactamente qué se escribe y cómo se estructura el proyecto. Con IA, el resultado depende en gran medida de qué tan bien se describe el problema: si la descripción es vaga o incompleta, el código generado puede no ser lo que se esperaba y hay que pedir correcciones varias veces hasta llegar al resultado deseado.

Por último, la dificultad de entrada es considerablemente menor cuando se usa IA. No es necesario tener todos los conocimientos de sintaxis ni recordar cómo funciona cada elemento. Sin IA, en cambio, es necesario tener una base sólida para poder arrancar y avanzar sin trabarse constantemente.

---

## Conclusión

Ningún enfoque es superior al otro en todos los casos. Desarrollar sin IA fomenta un aprendizaje más profundo y una comprensión real del código. Usar IA permite trabajar con mayor velocidad y explorar soluciones rápidamente. Lo ideal es conocer ambas formas y saber cuándo aplicar cada una según el contexto.
